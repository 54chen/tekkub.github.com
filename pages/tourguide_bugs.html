All bugs should be submitted to Google Code. Please look over the list first to make sure your issue isn't listed.

Please make sure you are set to receive email updates to tickets you open on GCode. If I post something that needs feedback from you, and you never reply, I will close the ticket out without a second thought! You can turn on notifications on the settings page.

Important info you should include

  • The version you are running
  • The version of related addons (Carto, TomTom) if the bug may relate to them
  • The guide (zone and level) you are using
  • The exact title of the objective (usually quest name) and the type (accept, complete, turnin, run...)
  • What you were doing when the bug occurred

Known conflicts

There are a number of "Quest Level" addons that append the quest's recommended level to the front of the text in the quest log and other places. A good number of these addons do this by hooking GetQuestLogTitle and modifying it's return values. This is a very bad practice. The addons listed below do this, they will break TG's auto-detection, and I do not intend to compensate for their bad code in TourGuide. If you want quest levels that don't interfere I've written Quel'evel as an alternative.

  • QuestLevel by Elkano (and the many "revived" versions floating around)
  • QuestLongList by Zevzabich
  • QuestIOn Can be disabled by turning off the quest level feature
  • Extended Quest Log 3

Submitting patches

I have had many problems with submitted patches, they never seem to be in the correct format. While I will not reject patches submitted to Google Code, I much prefer you fork my github repo and push changes directly. See below for details.

Hot forking action!

If you know how to use git, then I really shouldn't need to tell you how to get commits to me. Simply fork my repo on github and push to your new fork. You don't even have to contact me, I'll just pull the changes in as I see them come in!

If you don't know how to use git, here are a few great guides to get you started:

  • Using Git and Github on Windows
  • Using Git and Github on OSX (damn hippies)
  • How to fork a project to submit a patch

Please read How to create TourGuide data files before you submit files!
